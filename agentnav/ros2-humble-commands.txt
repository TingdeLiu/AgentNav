24. Auto recharge
Edit robot model and battery capacity:
Open auto_recharge_ros2/robot_info.yaml and select the robot model and battery capacity matching your robot

Use auto recharge:
① First map and save
ros2 launch slam_gmapping slam_gmapping.launch.py
ros2 launch wheeltec_nav2 save_map.launch.py
② Start navigation:
ros2 launch wheeltec_nav2 wheeltec_nav2.launch.py
③ Start auto recharge:
ros2 run auto_recharge_ros2 auto_recharge
In rviz, use topic "charger_position_update" to mark the charging dock position. Follow terminal prompts to proceed.

Other common commands

1. Build
Build all packages
colcon build
Note: Changes to launch files require a rebuild to take effect

Build all packages with symlink install (recommended)
colcon build --symlink-install
Note: Changes to launch files take effect without rebuilding

Build a specific package
e.g. build only turn_on_wheeltec_robot
colcon build --packages-select turn_on_wheeltec_robot [--symlink-install]

Limit parallel build threads to 2
colcon build --parallel-workers 2 [--symlink-install]

2. Recursively update modified time for all files in current directory:
find ./* -exec touch {} \;

3. Run in workspace to install all ROS package dependencies (rosdep is configured in the image):
rosdep install --from-paths src --ignore-src -r -y

4. Set system time:
sudo date -s "2022-06-15 09:00:00"

5. pip install using Douban mirror (much faster in China):
pip install -i https://pypi.doubanio.com/simple/ <package-name>

6. SSH login:
ssh -Y wheeltec@192.168.0.100

7. Grant executable permissions to all files in a folder:
sudo chmod -R 777 <folder>

8. Open map directory:
cd /home/wheeltec/wheeltec_ros2/src/wheeltec_robot_nav2/map
Manually save map:
ros2 run nav2_map_server map_saver_cli -f ~/map

View image topics with rqt:
rqt_image_view

View node and topic relationships:
rqt_graph

Generate TF tree PDF in current terminal path:
ros2 run tf2_tools view_frames

VNC adjust resolution:
xrandr --fb 1024x768
